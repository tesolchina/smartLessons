üìä Module 5: Data Visualization Best Practices

Presenter: Tian WU

üìà Choosing Appropriate Visualizations

Key Principle: Visualization choice depends on data type - categorical vs continuous variables require different approaches.

Categorical Variables

  • Problem: Scatter plots show disconnected dots for categorical data
  • Solution: Use bar charts with percentages
  • Example: Support level vs perceived fairness - show percentages of opposed/neutral/support across fairness categories

Continuous Variables

  • Best Practice: Use scatter plots for continuous variable relationships
  • Example: Distance to recycling facility vs support level

üì¶ Module 6: Box and Whisker Plots Explained

Presenter: Tian WU

üî¢ Five-Point Summary Statistics

Components of Box Plot:

  • Minimum: Smallest value in dataset
  • Q1 (25th percentile): Lower edge of box
  • Median (50th percentile): Middle line in box
  • Q3 (75th percentile): Upper edge of box
  • Maximum: Largest value within fence range

üéØ Interpreting Distribution Shape

Skewness Detection:
  • Right-skewed: Mean > Median (like salary distribution)
  • Outliers: Points beyond 1.5 √ó IQR from box edges
  • Box height: Shows middle 50% of data spread

Real-World Example: Hong Kong Salaries

Super-rich people inflate the average salary, but median better represents typical person's experience. Box plots help visualize this distinction between mean and median.

üîÑ Module 7: Introduction to Simulation Methods

Presenter: Tian WU

üöå Case Study: Bus Route 56 Efficiency Analysis

Simulation Objective: Evaluate efficiency of bus route adjustments using real data and mathematical modeling.

Background Context

  • Annual Process: 4 franchise bus companies submit route planning programs to Transport Department
  • Review Process: Department reviews and consults district councils
  • Implementation: 24-month temporary trial before permanent legislative changes

Bus 56 Specific Changes

  • Route: Extended distance (longer route)
  • Frequency: Increased from 2 to 3 buses per hour (morning)
  • Service Time: Added afternoon/night sessions and weekends
  • Puzzle: Original passenger load factor only 32% (criteria suggests 90% for increases)

üé≤ Three Random Components in Simulation

1. Travel Time Between Stops

  • Distribution: Normal distribution
  • Data Source: 30 trips of ETA data from HK government API
  • Implementation: np.random.normal(mean, std_dev)

2. Number of Waiting Passengers

  • Distribution: Poisson distribution
  • Parameter: Lambda (average arrivals per time unit)
  • Data Source: Field observations at bus stops

3. Passenger Alighting

  • Distribution: Binomial distribution
  • Parameter: Probability of getting off at each stop
  • Variation: Higher probability near MTR stations

üíª SimPy Framework

Four-Step Structure:

  1. Create Environment: simpy.Environment()
  2. Define Generator Function: Process logic with events
  3. Add to Environment: env.process(function)
  4. Run Simulation: env.run()

üìä Key Findings

Simulation Results:
  • Adjustment appears to have reduced overall seat utilization rate
  • Increased bus frequency means higher operational costs
  • Question: Does increased income cover additional costs?

Core Concept: Simulation uses mathematical models to approximate reality when we have limited data.

üîÑ The Simulation Process

  1. Choose Mathematical Models: Normal, binomial, Poisson distributions for different scenarios
  2. Collect Real Data: Field observations to feed into models
  3. Run Simulations: Generate approximations of reality
  4. Expert Consultation: Validate model choices with domain experts
